Arduino®;

Ce programme a été conçu pour fonctionner avec un Arduino®; Mega2560. Cependant, le code arduino (disponible ici est facilement modifiable et adaptable à la configuration souhaitée. Le nombre de capteur se règle grace à la constante appelée sensorNumber. Les entrées analogiques séléctionnées seront automatiquement les x premières entrées analogiques de l’Arduino®;.Une valeur de "debounce" est aussi paramétrée : il s’agit de la variable cycles. Elle permet de ne pas surcharger en message le port série de l’arduino. Plus le capteur est instable (transducteur piezo, viel interrupteur, ...) plus cette valeur devra être élevée. Une valeur seuil de signal (threshold) est aussi spéficiée afin de filter les signaux parasites qui pourraient être reçus sur les entrées. Elle doit impérativement rester comprise entre 0 ou rien n’est filtré, 1023 ou aucun signal ne passe. Enfin, si l’Arduino®; est programmé traiter plus de capteurs que ce qu’il y en a réellement branché, il est conseillé de relier les entrées non utilisées à la Terre (GND) pour empécher tout message parasite.

Midi

Pour fonctionner, ce programme a besoin d’utiliser un port midi virtuel. C’est par là que transiteront les messages entre le programme et l’application pilotable en midi.

Pour Linux

Pour créer des ports midi virtuels sur linux, il suffit d’éxécuter la commande suivante dans un terminal : sudo modprobe snd-virmidi snd_index=1. Vous pouvez vérifier la création de ces ports grâce à la commande cat /proc/asound/cards. Vous devriez normalement voir apparaitre une ligne VirMIDI - Virtual MIDI card 1. À partir de la, le programme sera capable de détecter ces ports midi virtuels et de s’y connecter.

Pour Mac OSX

Sous Mac OSX, il est possible de créer des ports virtuels midi à partir du menu "configuration audio et MIDI". Ces ports sont ensuite détéctés et peuvent être utilisés par le logiciel.

Pour Windows

Le logiciel LoopMIDI téléchargeable gratuitement sur internet permet de créer les ports virtuels midi necessaire au fonctionnement de ce logiciel sous Windows.

Fonctionnement

Initialisation

À l’ouverture de ce programme, il est demandé de configurer la connexion à l’Arduino®; en selectionnant le port de communication utilisé. Les choix par défaut sont normalement valable pour Windows ainsi que linux. Pour Mac OS, il faut impérativement compléter l’adresse du port : le port série à utiliser, est celui utilisé par l’ArduinoIDE. Il est aussi possible de trouver l’Arduino®; grâce à la commande ls /dev/tty.usbserial-*. Le port série ne peut pas être utilisé par 2 programmes simultanément, c’est pourquoi il est obligatoire de quitter tous les programmes utilisant le port série pour pouvoir utiliser ArduinoMidiBridge : il n’est par conséquent pas possible d’ouvrir 2 sessions simultanées d’ArduinoMidiBrigde avec un seul Arduino®;.

Il faut aussi sélectionner le périphérique midi virtuel à utiliser parmis la liste des périphériques utilisables.

Usage courant

L’usage du logiciel est très simple : dans le panneau du bas, on nomme un capteur, on choisi le canal arduino correspondant et le port MIDI auquel on veut le connecter. Une fois que toute ces informations sont complétées, on peut ajouter ce capteur aux capteurs actifs.Chaque capteur possède plusieurs attribut et réglages qui leur sont propre :

In : Il s’agit d’un pseudo vu mètre représentant le niveau d’entrée du canal arduino correspondant. Il permet de vérifier que l’on reçoit bien du signal de nos capteurs ainsi qu’avoir une idée de son intensité.

Préamplification : Plus qu’un réel préampli, il s’agit plutôt d’un rapport de multiplication du signal. Il est réglable grâce au slider ou en entrant la valeur manuellement. Pour être accéptée, la saisie doit être validée en appuyant sur la touche Entrée.

Min : Il s’agit de la valeur minimale du message midi envoyé. La saisie doit être validé avec la touche Entrée.

Max : Il s’agit de la valeur maximale du message midi envoyé. La saisie doit être validé avec la touche Entrée.

Out : Il s’agit du niveau de sortie de la derniere note MIDI envoyée. Elle se place sur une echelle de 1 à 127.

Bouton Mute : Cliquer sur ce bouton permet de muter la piste correspondante, c’est à dire que cela permet de ne plus envoyer de message avec cette piste de façon temporaire.

Bouton Solo : Cliquer sur le bouton solo d’une piste permet de l’isoler des autres et de ne plus emmettre des notes que par les pistes solo. Cependant, une piste solo peut ne pas emmettre de note si le bouton Mute All est activé. Si la piste est "muté", le solo surpasse le mute.

Bouton Impulsion : Ce bouton emmet une note midi à l’intensité maximale pendant une demi seconde. Il permet de "mapper" une piste à un effet. Il surpasse les conditions de Mute ou de Solo.

Bouton Supprimer : Ce bouton permet de supprimer la piste sélectionnée.

Bouton Mute All : Ce bouton a pour effet de muter toutes les pistes actives du logiciel. Il peut être utile pour mapper les capteurs sans recevoir des messages analogiques perturbateurs.
